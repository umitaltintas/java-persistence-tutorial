1 Ekosisteme Genel Bakis

Java'da veri tabani erisimi tek bir aractan ibaret degildir. Katmanli bir yapi soz konusudur:

+-------------------------------------------------------------+
|  GELISTIRICI KODU (Sen burada yaziyorsun)                    |
+-------------------------------------------------------------+
|  Spring Data JPA / Spring Data JDBC / Spring Data R2DBC      |  <- Ust duzey soyutlama
+-------------------------------------------------------------+
|  Jakarta Persistence (JPA) / Jakarta Data                    |  <- Standartlar (Kurallar)
+-------------------------------------------------------------+
|  Hibernate / EclipseLink                                     |  <- ORM Motorlari (Uygulayicilar)
+-------------------------------------------------------------+
|  JDBC / R2DBC                                                |  <- Dusuk seviye DB baglantisi
+-------------------------------------------------------------+
|  PostgreSQL / MySQL / Oracle                                 |  <- Veri Tabani
+-------------------------------------------------------------+

Katmanlar Arasi Iliski

JDBC

Java'nin veri tabanlariyla konusmak icin kullandigi en temel API. Tum ORM araclari sonucta JDBC uzerinden SQL calistirir.

JPA (Jakarta Persistence)

Bir spesifikasyon (kural seti). Tek basina is yapmaz. "Entity nasil tanimlanir, sorgu nasil yazilir" gibi kurallari belirler.

Hibernate

JPA spesifikasyonunu uygulayan (implement eden) bir kutuphane. Gercek isi yapan motordur. EclipseLink ise alternatifidir.

Spring Data JPA

Hibernate ve JPA uzerine bir soyutlama katmanidir. Gelistiriciyi tekrarlayan koddan kurtarir.

jOOQ

ORM degildir. SQL'i Java kodu icinde tip guvenli (type-safe) yazmayi saglar.

2 JDBC — Her Seyin Temeli

JDBC (Java Database Connectivity), Java'nin veri tabanina baglanmak icin sundugu en dusuk seviye API'dir. Her arac sonucta JDBC kullanir.

2.1 JDBC ile Basit Sorgu

import java.sql.*;

public class JdbcExample {

    public static void main(String[] args) throws Exception {
        // 1. Baglanti kur
        String url = "jdbc:postgresql://localhost:5432/mydb";
        Connection conn = DriverManager.getConnection(url, "user", "password");

        // 2. SQL hazirla (PreparedStatement ile SQL Injection korumasi)
        String sql = "SELECT id, name, price FROM products WHERE category = ?";
        PreparedStatement stmt = conn.prepareStatement(sql);
        stmt.setString(1, "ELECTRONICS");

        // 3. Calistir ve sonuclari oku
        ResultSet rs = stmt.executeQuery();
        while (rs.next()) {
            long id = rs.getLong("id");
            String name = rs.getString("name");
            BigDecimal price = rs.getBigDecimal("price");
            System.out.println(id + " - " + name + " - " + price);
        }

        // 4. Kaynaklari kapat
        rs.close();
        stmt.close();
        conn.close();
    }
}

2.2 JDBC'nin Sorunlari (Neden ORM'e Ihtiyac Duyuldu?)

Sorun Aciklama
Boilerplate Kod Her sorgu icin baglanti ac, statement hazirla, sonuc oku, kapat — tekrar tekrar
Tip Donusumu rs.getString(), rs.getLong() — her sutunu elle donusturmen gerekir
Nesne-Tablo Uyumsuzlugu SQL tablolari duz (flat), Java nesneleri hiyerarsik — donusum zor
Veritabani Bagimliligi Farkli DB'lerde SQL sozdizimi farkli olabilir

Bu sorunlar ORM (Object-Relational Mapping) kavramini dogurdu. ORM, Java nesneleri ile veri tabani tablolari arasindaki eslesmeyi otomatik hale getirir.

Bilgi Kontrolu

1. JDBC'de SQL Injection saldirisina karsi hangi yapi kullanilir?

Dogru! PreparedStatement, parametreleri guvenli sekilde baglar ve SQL Injection saldirilarina karsi koruma saglar.
Yanlis. Dogru cevap: B) PreparedStatement, SQL Injection'a karsi koruma saglar.

2. Asagidakilerden hangisi JDBC'nin sorunlarindan biri degildir?

Dogru! Otomatik nesne-tablo eslesmesi JDBC'de yoktur, bu bir sorun degil, eksikliktir. ORM araclari bu eksikligi giderir.
Yanlis. Dogru cevap: C) Otomatik nesne-tablo eslesmesi JDBC'de bulunmaz, bu bir sorun degil eksikliktir.

3. JDBC ile calisilindiginda kaynaklarin (Connection, Statement, ResultSet) kapatilmamasi ne gibi bir soruna yol acar?

Dogru! Kapatilmayan kaynaklar bellek sizintisina ve baglanti havuzunun tukenmesine yol acar.
Yanlis. Dogru cevap: A) Kapatilmayan kaynaklar bellek sizintisi ve baglanti havuzu tukenmesine yol acar.

3 JPA (Jakarta Persistence) — ORM Standardi

JPA bir spesifikasyondur, kutuphane degildir. "Java'da ORM nasil yapilir" sorusunun kurallarini belirler. Eski adi Java Persistence API, yenisi Jakarta Persistence'dir.

3.1 Entity Tanimlama

JPA'da bir Java sinifini veri tabani tablosuna eslemek icin anotasyonlar kullanilir:

import jakarta.persistence.*;
import java.math.BigDecimal;
import java.time.LocalDateTime;

@Entity                          // Bu sinif bir veri tabani tablosuna karsilik gelir
@Table(name = "products")       // Tablo adi (yazilmazsa sinif adi kullanilir)
public class Product {

    @Id                          // Birincil anahtar (Primary Key)
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // DB otomatik artan ID uretir
    private Long id;

    @Column(nullable = false, length = 200)  // NOT NULL, max 200 karakter
    private String name;

    @Column(precision = 10, scale = 2)  // Ondalikli sayi: 10 basamak, 2'si ondalik
    private BigDecimal price;

    @Column(columnDefinition = "TEXT")   // Uzun metin
    private String description;

    @Column(name = "created_at")         // Sutun adi Java'daki field adindan farkliysa
    private LocalDateTime createdAt;

    @Enumerated(EnumType.STRING)         // Enum'u string olarak DB'ye yaz
    private ProductStatus status;

    // Getter ve Setter'lar
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public BigDecimal getPrice() { return price; }
    public void setPrice(BigDecimal price) { this.price = price; }

    public String getDescription() { return description; }
    public void setDescription(String description) { this.description = description; }

    public LocalDateTime getCreatedAt() { return createdAt; }
    public void setCreatedAt(LocalDateTime createdAt) { this.createdAt = createdAt; }

    public ProductStatus getStatus() { return status; }
    public void setStatus(ProductStatus status) { this.status = status; }
}

Temel JPA Anotasyonlari

@Entity — Sinifi bir veri tabani tablosuna esler.
@Table — Tablo adini belirtir (yazilmazsa sinif adi kullanilir).
@Id — Birincil anahtari (Primary Key) isaretler.
@GeneratedValue — ID uretim stratejisini belirler.
@Column — Sutun ozelliklerini tanimlar (nullable, length, precision vb.).
@Enumerated — Enum degerlerinin DB'ye nasil yazilacagini belirler.

3.2 Iliski (Relationship) Tanimlama

@Entity
@Table(name = "orders")
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String status;

    // Bir siparisin birden fazla kalemi olabilir (One-to-Many)
    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<OrderItem> items = new ArrayList<>();

    // Bir siparisin bir kullanicisi vardir (Many-to-One)
    @ManyToOne(fetch = FetchType.LAZY)   // LAZY: Kullanici bilgisi istenene kadar yukleme
    @JoinColumn(name = "user_id")
    private User user;
}

@Entity
@Table(name = "order_items")
public class OrderItem {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private int quantity;
    private BigDecimal unitPrice;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "order_id")
    private Order order;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "product_id")
    private Product product;
}

Iliski Tipleri

@OneToMany — Bir nesnenin birden fazla alt nesnesi olabilir (ornegin bir siparisin bircok kalemi).
@ManyToOne — Bircok nesne tek bir ust nesneye bagli olabilir.
mappedBy — Iliskinin sahibi olmayan tarafi belirtir.
cascade = CascadeType.ALL — Ana nesne uzerindeki islemler alt nesnelere de uygulanir.
orphanRemoval = true — Listeden cikarilan alt nesneler DB'den de silinir.
FetchType.LAZY — Iliskili veri, ihtiyac duyulana kadar yuklenmez (performans icin onemli).

Dikkat: FetchType.LAZY kullanildiginda, iliskili veriye @Transactional kapsami disinda erismeye calismak LazyInitializationException hatasina neden olur. Iliskili verilere her zaman aktif bir transaction icinde erisin.

3.3 JPQL (Java Persistence Query Language)

JPA kendi sorgu dilini sunar. SQL'e benzer ama tablo yerine Java sinifi adlarini kullanir:

// SQL:  SELECT * FROM products WHERE price > 100
// JPQL: SELECT p FROM Product p WHERE p.price > 100

// SQL:  SELECT p.name, COUNT(oi.id)
//       FROM products p JOIN order_items oi ON p.id = oi.product_id
//       GROUP BY p.name
// JPQL:
@Query("SELECT p.name, COUNT(oi) FROM Product p JOIN p.orderItems oi GROUP BY p.name")
List<Object[]> getProductOrderCounts();

JPQL ve SQL Arasindaki Fark

JPQL, SQL'e cok benzer ancak temel bir farki vardir: SQL tablo ve sutun adlarini kullanirken, JPQL Java sinif ve alan (field) adlarini kullanir. Bu sayede veri tabani bagimsiz sorgular yazilabilir. Ornegin SQL'de SELECT * FROM products yazilirken, JPQL'de SELECT p FROM Product p yazilir.

2. JPQL'de SQL'den farkli olarak ne kullanilir?

Dogru! FetchType.LAZY, iliskili verinin ihtiyac duyulana kadar yuklenmemesini saglar. Bu, performans icin onemlidir.
Yanlis. Dogru cevap: B) LAZY fetch stratejisi, iliskili verinin ihtiyac duyulana kadar yuklenmemesini saglar.
